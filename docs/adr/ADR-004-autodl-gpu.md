# ADR-004: GPU 推理使用 AutoDL 按需租用

- 日期：2026-03-09
- 状态：已确认

---

## 背景

Calliope 的 Python 推理服务需要 GPU 来运行 AudioCraft MusicGen。GPU 的获取方式直接决定推理成本和运维复杂度。项目初期是个人练习项目，预算极为有限（目标 ¥0~150/月）。

---

## 决策

**GPU 推理使用 AutoDL 按需租用（RTX 3090，¥2/小时），阶段 0（无 GPU 时）使用 SiliconFlow API 替代。**

---

## 备选方案

| 方案 | 价格 | 优点 | 缺点 |
|------|------|------|------|
| **AutoDL 按需** | ¥2/h（RTX 3090，基于 2026-03 定价）| 国内，支付宝，按秒计费，最便宜 | 需要手动启停；冷启动 1-3 分钟 |
| **SiliconFlow API（阶段0）** | 注册送额度，用完付费 | 零运维，无需 GPU；国内 | 不自托管 AudioCraft，质量取决于平台；有 API 调用限制 |
| **AWS EC2 p3.2xlarge** | ~$3/h（约 ¥22/h）| 弹性伸缩成熟 | 需要国际信用卡；国内访问慢；价格是 AutoDL 的 10 倍 |
| **GCP / Azure GPU** | $3-5/h | 同上 | 同上 |
| **自购二手 GPU** | ¥3000~8000（RTX 3090 二手） | 无持续费用 | 需要本地机器；电费；搬迁困难；个人项目不划算 |
| **Vast.ai / Lambda Labs** | $0.5-1/h | 较便宜 | 需要国际信用卡；网络延迟；稳定性一般 |
| **矩池云 / 趋动云** | ¥1.5-3/h | 国内，类 AutoDL | AutoDL 市场份额更大，社区资料更多 |

---

## 理由

1. **成本最优（基于 2026-03 定价）**：AutoDL RTX 3090 约 ¥2/小时，是 AWS 同级 GPU 价格的 1/10。对于个人练习项目，这是决定性因素。

2. **国内无障碍（基于 2026-03 平台现状）**：支付宝付款，无需国际信用卡，注册即用。国内用户（目标用户也在国内），网络延迟低，GPU 到七牛云 OSS 的上传速度快。

3. **按需启停策略与冷启动影响**：推理服务不是 24 小时运行的，通过检测 Redis Stream 队列是否有待处理消息来决定是否保持运行：
   - 有任务时：启动 GPU 实例（或保持运行）
   - 队列空闲超过 10 分钟：关闭 GPU 实例
   - 每月实际使用估算（10 个测试用户，每人 5 次/天）：约 50 次/天 × 2 分钟/次 = 100 分钟/天 ≈ ¥3.3/天 ≈ ¥100/月
   - **冷启动与 NFR 关系**：冷启动 1-3 分钟发生在 Worker 取到任务之前（`started_at` 之前），不计入 NFR 的 180s 推理超时窗口。用户端总等待时间在 MVP 内测阶段无硬性 SLA，前端通过队列位置和进度提示缓解体感。降级方向：若冷启动超时率持续偏高，可将最小保活时长从 10 分钟延长，或预热首个任务时向用户提示"GPU 启动中，预计 X 分钟"。

4. **阶段 0 降级方案**：初期无需 GPU 的情况下，Python 推理服务通过 `InferenceBackend` 接口切换为调用 SiliconFlow 的音乐生成 API，实现零成本验证整体链路。代码层面通过环境变量 `INFERENCE_BACKEND=siliconflow|musicgen` 切换，无需修改业务代码。

5. **社区资料（基于 2026-03 平台份额）**：AutoDL 是国内最大的 GPU 租用平台，有大量 AudioCraft 部署教程，遇到问题容易找到解决方案。

---

## 后果

**优点：**
- 成本极低（¥0~150/月，视使用量）
- 无需国际信用卡，国内开发者友好
- 通过 `InferenceBackend` 接口抽象，阶段 0 可完全不依赖 GPU

**缺点/风险：**
- **冷启动延迟**：GPU 实例从停止到可用约需 1-3 分钟，期间任务在 Redis 队列中等待。对于内部测试场景可接受。
- **成本不可预测**：如果公开给外部用户，每日配额（5次/用户）+ 队列上限（20个）是主要保护机制，需要严格执行。
- **单点故障**：单 GPU 实例无高可用，故障时任务积压。内部测试阶段可接受（非功能性需求：GPU 无 SLA 保证）。

**GPU 成本控制机制（必须实现）：**
1. 每用户每日上限 5 次（credits 表）
2. 队列最大积压 20 个（超出返回 QUEUE_FULL）
3. GPU 空闲 10 分钟自动停机脚本
4. 月度成本告警（AutoDL 控制台设置预算提醒）

---

## 复审触发条件

出现以下情况时复审本决策：

| 触发条件 | 建议动作 |
|---------|---------|
| 月成本连续超预算（¥150）或使用量大幅增长 | 评估预留实例或切换更低成本方案 |
| 冷启动导致用户投诉或任务超时率升高 | 延长最小保活时长，或评估轻量级预热机制 |
| AutoDL 供给不稳定（抢不到机器、频繁掉线） | 评估矩池云/趋动云等备选平台（见备选方案表） |
| 项目进入公开运营阶段，需要 SLA 保证 | 评估云厂商托管 GPU（AWS/阿里云 GPU 实例）或自购 GPU |
| 推理规模增长到需要多 GPU 并行 | 重新评估 AutoDL 多实例方案或迁移到弹性 GPU 集群 |
