# Calliope 项目主计划

> 每次开启新的 Claude 对话时，将本文档贴给 Claude 作为上下文。
> 每完成一个任务或做出重要决策后，及时更新本文档。

---

## 项目概述

**目标：** 构建一套模仿 Suno/Udio 的 AI 音乐生成系统
**附加目标：** 项目发起人（Android 架构师）通过此项目成长为后端/系统/AI 架构师
**输出目标：** 将项目过程沉淀为系列技术文章，发布到掘金/CSDN

**支持平台：** Android（原生 Kotlin）、iOS（原生 Swift）、H5（原生 JS）
**后端语言：** Go（API 层）+ Python（AI 推理层）
**上线策略：** 个人练习项目，暂不公开上线；架构按高并发设计，部署按最小成本执行

---

## 技术选型（已确认）

| 层 | 选型 | 备注 |
|--|--|--|
| API 层 | Go + Gin → 后期 gRPC | 刻意练习，非最优捷径 |
| 推理层 | Python + FastAPI + AudioCraft | 与 AI 生态统一 |
| 数据库 | MySQL | 团队熟悉度优先 |
| 缓存/队列 | Redis | 早期单机，后期集群 |
| 对象存储 | 七牛云（10GB 永久免费）| 国内，无需国际信用卡 |
| CDN | 七牛云 CDN | 含免费额度 |
| GPU 推理 | AutoDL 按需租用 | 国内，支付宝，最便宜 |
| 阶段0推理 API | 硅基流动 SiliconFlow | 注册送额度，替代自建 GPU |
| 容器化 | Docker → K8s（第三阶段） | 渐进式引入 |
| 监控 | Prometheus + Grafana | 第三阶段引入 |

## 阶段 3：编码实现（TDD）

**目标：** 按模块逐步实现，测试先行
**Claude 使用方式：** Plan Mode（每个模块实现前）→ 确认 → 执行

### 实现顺序（依赖关系决定）

```
基础设施层
├── [ ] 数据库连接和 migration
├── [ ] Redis 连接
└── [ ] 对象存储客户端

核心业务层
├── [ ] 用户认证（注册/登录/JWT）
├── [ ] 音乐任务管理（创建/查询/取消）
├── [ ] 任务队列（生产者/消费者）
└── [ ] 音频文件管理（上传/下载/删除）

AI 推理层（Python）
├── [ ] FastAPI 服务骨架
├── [ ] AudioCraft 集成
├── [ ] 任务消费 Worker
└── [ ] 推理结果回写

客户端层
├── [ ] Android：网络层 + 音频播放
├── [ ] iOS：网络层 + 音频播放
└── [ ] H5：API 调用 + 音频播放

实时通信
└── [ ] WebSocket 任务进度推送
```

### 每个模块的实现流程

```
1. 写测试用例（Claude 辅助）
2. Plan Mode：Claude 规划实现方案
3. 确认方案
4. 实现代码
5. 跑测试
6. Code Review（Claude 多视角 review）
7. 更新本文档
```

### 完成标准

> 单元测试覆盖率 > 70%，核心业务逻辑 100% 覆盖。所有接口与 OpenAPI 文档一致。
